## Writing Clean Code

Clean code is code that is easy to understand and easy to change. It's not just about making the computer understand your instructionsâ€”it's about making your intentions clear to other humans (including your future self).[^1]

[^1]: Robert C. Martin, "Clean Code: A Handbook of Agile Software Craftsmanship," Prentice Hall, 2008.

### Key Principles

**Meaningful Names**: Variables, functions, and classes should have names that reveal their intent. A name should tell you why it exists, what it does, and how it's used.

**Small Functions**: Functions should do one thing. They should do it well. They should do it only. Large functions with multiple responsibilities are hard to understand and test.

**Comments**: The best comment is the comment you found a way not to write. When you must comment, make sure it explains the _why_, not the _what_.

**Error Handling**: Error handling is important, but it shouldn't obscure the main logic. Keep your error handling separate and clean.
